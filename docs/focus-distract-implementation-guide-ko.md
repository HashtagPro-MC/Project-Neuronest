# 집중/산만 판별 + 코인 모터 + 앱 알림 구현 가이드 (초보자용)

이 문서는 **센서 기반 집중/산만 판별 시스템**을 단계별로 구현하는 실전 가이드입니다.
목표는 아래 3가지를 안정적으로 연결하는 것입니다.

1. 센서로 집중/산만 상태 판별
2. 산만 상태일 때 코인 모터 진동
3. 상태/특징 데이터를 앱으로 전송하고 앱 알림 표시

---

## 0) 먼저 정리: 전체 구조

```text
MAX30102 + MPU6050
        ↓
ESP32에서 40ms 샘플링 + 3초 윈도우 특징 계산
        ↓
집중/산만 판별 로직 (규칙 기반 또는 모델 기반)
        ↓
(산만이면) 코인 모터 진동
        ↓
BLE/Wi-Fi로 앱에 데이터 전송
        ↓
iOS 앱에서 대시보드/로컬 알림 표시
```

---

## 1) 하드웨어 준비 (실패를 줄이는 핵심)

- MCU: ESP32 (BLE 사용 권장)
- 심박: MAX30102
- 모션: MPU6050
- 피드백: 코인 진동 모터 + 트랜지스터/MOSFET + 다이오드

### 배선 체크 포인트
- MAX30102/MPU6050: I2C (`SDA`, `SCL`) 공유 가능
- 코인 모터: **GPIO에 직접 연결하지 말고** 드라이버 회로 사용
- GND 공통 연결
- 전원 노이즈 줄이기 위해 모터 쪽에 디커플링 권장

> 모터를 GPIO에 직접 물리면 리셋/오동작/포트 손상이 날 수 있습니다.

---

## 2) 펌웨어 베이스 만들기

### 샘플링/윈도우 권장값
- 샘플링 주기: `40ms` (25Hz)
- 윈도우: `3초` (`75샘플`)
- 초기 베이스라인: `120초`

### 계산할 특징(Feature)
- `motionMean`, `motionVar` (움직임 평균/분산)
- `bpmMean`, `bpmVar` (심박 평균/분산)
- `rmssd` (심박변이도)
- `jerkMean` (움직임 변화량)

### 안정성 규칙
- 심박 감지 실패 시 이전 유효값 유지
- RR interval 유효 개수 부족하면 `rmssd` 계산 보류
- 센서 초기화 실패 시 재시도 또는 명확한 오류 출력

---

## 3) 라벨 데이터 수집 단계 (가장 중요)

모델 정확도는 수집 데이터 품질로 결정됩니다.

### 라벨 정책
- `1 = Focused`
- `0 = Distracted`
- 필요하면 `2 = Neutral`

### 수집 방법
1. 시리얼 명령으로 라벨 전환 (`f/d/n`)
2. 최소 20~30분 이상 수집
3. 다양한 상황 포함
   - 조용한 책상 집중
   - 휴대폰 확인/고개 움직임 많은 상태
   - 피로 상태
4. CSV로 저장

### 권장 컬럼
```text
timestamp,motion_z,motion_var_z,bpm_z,bpm_var_z,rmssd_z,jerk_z,label
```

---

## 4) 판별 로직 1차 구현 (규칙 기반)

처음부터 AI 모델을 올리기보다, 규칙 기반 임계치로 시작하면 디버깅이 쉽습니다.

예시:
- `motion_z > T1` 이고 `jerk_z > T2` 이면 산만
- `rmssd_z` 급감 + `bpm_z` 급상승이면 산만 가중치 추가

### 디바운스(오탐 방지)
- 1회 판정으로 즉시 상태 변경하지 말고
- `N`개 연속 윈도우에서 산만일 때만 산만 확정

권장:
- `N=2~3`
- 상태가 바뀐 후 최소 쿨다운 `5~10초`

---

## 5) 판별 로직 2차 구현 (모델 기반)

데이터가 충분해지면 앱/서버에서 간단한 모델로 시작합니다.

### 추천 시작 모델
- Logistic Regression
- Random Forest

### 절차
1. CSV 정리 (결측/이상치 제거)
2. train/validation/test 분할
3. F1-score, recall(산만 클래스) 확인
4. 임계치 튜닝 (정밀도 vs 재현율)
5. 최종 임계치만 펌웨어에 반영하거나 앱에서 판별

---

## 6) 코인 모터 동작 설계

### 트리거 정책
- 산만 확정 시 진동 패턴 1회
- 예: `150ms ON → 100ms OFF → 150ms ON`

### 과진동 방지
- 최소 재트리거 간격(`motorCooldownMs`) 설정 (예: 8초)
- 산만 상태가 계속돼도 무한 진동 금지

### 안전성
- 저전압/배터리 부족 시 진동 강도/횟수 제한

---

## 7) 앱 데이터 전송 설계

BLE 기준 권장 payload:

```json
{
  "ts": 1730000000,
  "state": "DISTRACTED",
  "score": 0.82,
  "motion_z": 1.2,
  "bpm_z": 0.6,
  "rmssd_z": -0.4,
  "battery": 78
}
```

### 전송 주기
- raw 센서는 전송량 큼 → 특징/상태 중심으로 1~3초 주기 전송 권장

### 연결 안정화
- 연결 끊김 시 최근 N개 버퍼 저장 후 재연결 시 재전송
- 패킷에 시퀀스 번호 추가

---

## 8) iOS 앱 알림/화면

### UI 최소 구성
- 현재 상태: 집중/산만
- 집중 점수(0~100)
- 최근 5분 추이 그래프
- 진동 히스토리

### 알림 정책
- 같은 알림 연속 발생 제한 (예: 1분당 최대 1회)
- 메시지 예시: “집중이 흔들리고 있어요. 20초 호흡 후 다시 시작해볼까요?”

---

## 9) 서버 연동(선택)

앱에서 백엔드로 세션 로그 업로드:
- 세션 시작/종료 시점
- 산만 이벤트 빈도
- 시간대별 집중도

이 데이터로 개인화 임계치(아침/밤) 적용 가능

---

## 10) 단계별 체크리스트

### 단계 A: 센서 안정화
- [ ] 10분 이상 연속 측정에서 값 튐/리셋 없음
- [ ] 손가락 미착용 시 심박 오탐 낮음

### 단계 B: 판별 안정화
- [ ] 산만 상황 재현 시 70% 이상 탐지
- [ ] 집중 상황에서 오탐 10% 이하

### 단계 C: 피드백 안정화
- [ ] 산만 확정 시 1초 내 진동
- [ ] 쿨다운 동작 정상

### 단계 D: 앱 안정화
- [ ] BLE 재연결 자동 복구
- [ ] 알림 스팸 없음
- [ ] 데이터 누락률 허용범위 이내

---

## 11) 가장 많이 나는 문제와 예방

1. **심박 신호 불안정**
   - 센서 착용 위치 고정, 광량/압착 일관성 유지
2. **모터 때문에 MCU 리셋**
   - 드라이버 회로 + 전원 분리/보강
3. **산만 오탐 과다**
   - 단일 임계치 대신 다중 특징 + 디바운스 적용
4. **앱 알림 과다**
   - rate-limit + 쿨다운 적용

---

## 12) 빠르게 시작하는 추천 순서

1. 센서값 시리얼 출력 안정화
2. 3초 윈도우 특징 계산
3. 라벨 데이터 수집
4. 규칙 기반 산만 판정 + 모터 진동
5. BLE로 상태 전송
6. 앱 대시보드 + 알림
7. 모델 학습으로 정확도 개선

---

## 13) 구현 기준(권장 기본값)

- 샘플링: 25Hz
- 윈도우: 3초
- 상태 확정: 연속 2윈도우
- 모터 쿨다운: 8초
- 앱 알림 쿨다운: 60초
- 베이스라인 재학습: 하루 1회 또는 세션 시작 2분

이 기본값으로 시작하면 시행착오를 줄이면서 빠르게 동작하는 MVP를 만들 수 있습니다.
