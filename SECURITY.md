# Security Policy

## Supported Versions
We support security fixes for the latest version on the `main` branch.

## Reporting a Vulnerability
If you discover a security vulnerability, please **do not** open a public GitHub issue.

Instead, report it privately by:
1) Opening a GitHub Security Advisory (preferred), or  
2) Emailing the maintainers at: **security@YOURDOMAIN.com** (replace)

Please include:
- A clear description of the issue
- Steps to reproduce
- Affected component (iOS app / ESP32 bridge / Arduino firmware)
- Any proof-of-concept code or logs (if available)

## Response Timeline
We aim to:
- Acknowledge within **72 hours**
- Provide a mitigation plan or fix timeline within **7 days**, when possible

## Coordinated Disclosure
We appreciate responsible disclosure. Please allow time for a fix before sharing details publicly.
